# Escape Group LINE Bot — 產品設計文件

> 2026-03-30 — 基於產品重新定位討論 + 問卷調查（15 份回覆）

## 產品定位

**從「獨立密室逃脫揪團網站」轉向「LINE Bot 密室揪團工具」**

為一個 100+ 人的密室逃脫 LINE 社群打造的私人揪團工具。不是公開平台，只服務該群組成員。

### 核心問題（問卷驗證）

1. **開團訊息被洗掉**（10/15）— 最大痛點
2. **湊不齊人**（8/15）— 不知道誰有興趣
3. **搜尋/篩選困難** — 無法快速找到想加入的團

### 不是主要問題

- 放鴿子（11/15 幾乎沒遇過）— 簡單記錄即可，不需要複雜的信用系統

## 核心概念

**一種團，兩個入口：**

- **開團**（私訊 Bot）→ 填表單選密室（必填）→ 設時間/人數 → Bot 發公告
- **找團**（私訊 Bot）→ 依日期/地區/關鍵字搜尋 → 加入

## 三個場景

| 場景         | 角色                | 互動方式                             |
| ------------ | ------------------- | ------------------------------------ |
| **個人私訊** | 管理中心 + 開團入口 | Rich Menu 按鈕，LIFF 表單            |
| **大群**     | 公佈欄              | Bot 只發公告，不接受指令，不回覆訊息 |
| **成團小群** | 行程助手 + 協調     | 提醒、出席確認                       |

## 功能規格

### P0 — 核心功能

#### 1. 開團

- 私訊 Bot → 點 Rich Menu「開團」→ 開啟 LIFF 表單
- 表單欄位：
  - 密室名稱（必填）
  - 工作室
  - 地區（北/中/南/東）
  - 活動時間（可之後再定）
  - 人數上限
  - 已有人數（含團主，給私下已找好的人用）
- 建立後：
  - Bot 立即在大群發一張 Flex Message 公告卡片
  - 每日固定時段（如晚上 8 點）發彙整卡片時也會包含此團

#### 2. 瀏覽/搜尋所有團

- 私訊 Bot → 點 Rich Menu「找團」
- 列出所有開放的團，**依活動日期排序（最早的在最前面）**
- 篩選條件：
  - 地區（北/中/南）
  - 時間範圍（我哪天有空）
  - 關鍵字搜尋（密室名稱、工作室名稱）
- 呈現方式：Flex Message 卡片列表，可捲動
- 類似行事曆的概念：日期分組，每天下面列出該天的團

#### 3. 加入團

- 在大群公告卡片上點「我要加入」→ 跳到私訊確認
- 或在搜尋結果中點「加入」
- 加入後團的「目前人數」更新，大群公告不即時更新（等彙整）

### P1 — 重要功能

#### 4. 每日彙整公告

- 每天固定時段（如 20:00）Bot 在大群發一張彙整卡片
- 列出所有目前開放的團，依日期排序
- 標註「還差 N 人」「即將額滿」等狀態
- 大家養成習慣知道固定時間會有更新

#### 5. 活動前自動提醒

- 活動前一天 + 當天早上，Bot 私訊所有成員提醒
- 內容：密室名稱、時間、地點、成員名單

#### 6. 標記已確認人數

- 開團時可以填「已有 N 人」，公告顯示「3/6 人（還差 3 人）」
- Bot 只管透過平台加入的人，私下找的人團主自己負責

### P2 — 次要功能

#### 7. 退團（跳車）

- 成員私訊 Bot 申請退團 → Bot 通知團主
- 團主同意 → 成員移出、名額釋放
- 團主拒絕 → 成員收到通知

#### 8. 成團後建群

- 人齊了 → Bot 私訊團主「請建立 LINE 群組/OpenChat，貼回邀請連結」
- 團主貼連結 → Bot 轉發給所有成員
- Bot 被邀請進群後自動接管（行程摘要、提醒、出席確認）

### P3 — 後續擴充

#### 9. 出席紀錄

- 活動結束後，團主回報誰沒到
- Bot 記錄出席/缺席次數
- 只有超過一定次數才顯示（問卷 7/15 支持此方案）
- 任何人可查詢某人的紀錄

#### 10. 訂閱通知

- 訂閱特定密室/工作室/地區
- 有新團符合 → Bot 主動私訊通知

## 存取控制

- Bot 透過 LINE API 取得大群成員列表
- 私訊 Bot 時檢查 userId 是否在大群中
- 不在大群的人 → Bot 拒絕服務

## 管理員

- 手動指定（寫在環境變數中）
- 管理員可以：管理黑名單（未來）、查看統計

## 互動方式

- **Rich Menu**（底部固定選單）：開團、找團、我的團、我的紀錄
- **LIFF 表單**：開團填寫（一頁完成，比一問一答快）
- **Flex Message 按鈕**：加入團、查看詳情
- **12/15 使用者要求按鈕/選單為主**，不使用指令

## 資料模型

```
users
  ├── id
  ├── lineUserId
  ├── displayName
  ├── pictureUrl
  ├── attendCount
  ├── noShowCount
  ├── createdAt

groups
  ├── id
  ├── hostId            → users.id
  ├── roomName          — 密室名稱（必填）
  ├── studio            — 工作室
  ├── location          — 地區
  ├── datetime          — 活動時間（可選）
  ├── maxMembers        — 人數上限
  ├── prefilledMembers  — 已確認人數（私下找的）
  ├── status            — open / full / completed / cancelled
  ├── lineGroupId       — 成團後的群組 ID
  ├── lastAnnouncedAt   — 上次公告時間
  ├── createdAt

group_members
  ├── id
  ├── groupId           → groups.id
  ├── userId            → users.id
  ├── status            — accepted / attended / no_show
  ├── joinedAt

subscriptions
  ├── id
  ├── userId            → users.id
  ├── type              — room / studio / location
  ├── value             — 訂閱的值
  ├── createdAt
```

## 技術架構

```
LINE App
  ├── 大群（Bot 發公告，不互動）
  ├── 私訊（Rich Menu + LIFF 表單）
  └── 成團小群（Bot 被邀請後接管）

        ↕ LINE Messaging API (Webhook)

Bot Server（Node.js / TypeScript 或 Python）
  ├── Webhook Handler
  ├── LIFF 頁面（開團表單）
  ├── 業務邏輯（開團/加入/查詢/訂閱）
  └── Cron Job（每日彙整公告、活動提醒）

        ↕ Drizzle ORM（或對應 ORM）

Neon PostgreSQL（Serverless DB）
```

- **部署**：單一專案，Vercel Functions / Cloudflare Workers / Railway
- **無 Docker 依賴**：本地開發直接連 Neon dev branch
- **無 monorepo**：單一專案結構

## 問卷原始數據摘要

- 回覆數：15
- 最大痛點：訊息被洗掉（67%）、湊不齊人（53%）
- 放鴿子：73% 幾乎沒遇過
- 最想要功能：地區/時間篩選（80%）、避免撞期（73%）、瀏覽所有團（67%）、自動提醒（60%）
- 互動方式：80% 要按鈕/選單
- 大群行為：53% 希望固定時段統一發
- 開放題：有人提到 99script.cc（通用工具，非社群特化）；多人要求行事曆式瀏覽 + 關鍵字搜尋 + 日期排序

## 與 99script.cc 的差異

- **99script.cc**：通用揪團網站，需要離開 LINE
- **我們**：LINE Bot 原生體驗，為特定社群量身打造，不用離開聊天室

## 與現有 Web App 的關係

現有的 SvelteKit web app 將被取代。唯一保留的 web 元素是 LIFF 頁面（開團表單），運行在 LINE 內建瀏覽器中。
